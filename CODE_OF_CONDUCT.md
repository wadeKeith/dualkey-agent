# Code of Conduct

## Our Pledge

We want DualKey to be a project where contributors and users can collaborate without harassment. We expect participation to stay respectful, technically direct, and focused on improving the project.

## Expected Behavior

- Be respectful in issues, pull requests, reviews, and discussions.
- Critique code and ideas directly, without attacking people.
- Prefer evidence, repro steps, and concrete examples over rhetoric.
- Assume good intent until there is evidence otherwise.
- Respect reasonable privacy and security boundaries when sharing logs, receipts, policies, or traces.

## Unacceptable Behavior

- Harassment, discrimination, or intimidation.
- Personal attacks, insults, or sustained hostile behavior.
- Publishing private credentials, secrets, or sensitive operational data without permission.
- Deliberate spam, trolling, or derailing technical discussions.

## Scope

This Code of Conduct applies to project spaces, including the repository, issue tracker, pull requests, discussions, and any project-managed chat or release process.

## Reporting

If you need to report a conduct issue, open a private security-style report if sensitive material is involved, or contact the maintainers through the repository's published support path.

## Enforcement

Project maintainers may remove comments, close threads, reject contributions, or block participants whose behavior violates this policy.
